# 도메인 이벤트 설계: TransactionalEventListener

[[기술적-의사결정]]

---

## 문제: 가입 시 할 일 자동 생성

신규 회원 가입 시 당일 할 일 3개를 자동 생성해야 합니다.
`MemberService`가 `TodoService`를 직접 호출하면 두 도메인이 강하게 결합됩니다.

```
// 나쁜 예 — member 도메인이 home 도메인을 직접 의존
class MemberService(
    private val todoService: TodoService,  // ← 크로스 도메인 직접 의존
) {
    fun findOrCreateMember(...) {
        val member = memberRepository.save(...)
        todoService.generateDailyTodos(member, today)  // ← 강결합
    }
}
```

---

## 결정: Spring ApplicationEvent + AFTER_COMMIT

```kotlin
// MemberService — 이벤트 발행 (TodoService를 전혀 모름)
eventPublisher.publishEvent(MemberRegisteredEvent(member))

// MemberRegistrationEventListener — 이벤트 수신 (member 도메인을 모름)
@TransactionalEventListener(phase = TransactionPhase.AFTER_COMMIT)
fun onMemberRegistered(event: MemberRegisteredEvent) {
    val today = LocalDate.now(ZoneId.of("Asia/Seoul"))
    todoService.generateDailyTodos(event.member, today)
}
```

두 도메인은 `MemberRegisteredEvent`라는 계약만 공유하고, 서로의 구현을 알지 못합니다.

---

## AFTER_COMMIT vs 동일 트랜잭션

| 페이즈 | 동작 | 언제 쓰는가 |
|--------|------|------------|
| `@EventListener` (기본) | 발행자 트랜잭션 **안에서** 실행 | 원자성이 필요할 때 (함께 성공·실패) |
| `AFTER_COMMIT` | 발행자 트랜잭션 **커밋 후** 별도 실행 | 수신자 실패가 발행자 롤백으로 번지면 안 될 때 |

### 이 경우 AFTER_COMMIT을 선택한 이유

만약 같은 트랜잭션 안에서 Todo를 생성하다 실패하면 회원 등록까지 롤백됩니다.
"할 일 생성 실패 → 회원 가입 실패"는 비즈니스적으로 허용되지 않습니다.

`AFTER_COMMIT`을 쓰면 할 일 생성이 실패해도 회원은 이미 커밋된 상태이므로 가입은 보존됩니다.
Todo가 빠진 경우는 `TodoScheduler`가 다음 00:00에 멱등 생성으로 보완합니다.

## 이벤트 클래스 위치 규칙

```
domain/member/event/MemberRegisteredEvent.kt            ← 발행 도메인에 위치
domain/home/service/MemberRegistrationEventListener.kt  ← 처리 도메인에 위치
```

이벤트 클래스는 *발행하는* 도메인에, 리스너는 *처리하는* 도메인에 배치했습니다.
어느 도메인이 이벤트를 발행하는지를 패키지만 봐도 알 수 있습니다.
`doc/events/README.md`에 전체 이벤트 레지스트리를 관리합니다.

## 전체 이벤트 목록

| 이벤트 | 발행 | 리스너 | 페이즈 |
|--------|------|--------|--------|
| `MemberRegisteredEvent` | `MemberService.findOrCreateMember()` | `MemberRegistrationEventListener` | `AFTER_COMMIT` |

## 트레이드오프

| 항목 | 내용 |
|------|------|
| 할 일 생성 실패 시 | 가입 당일 할 일 없음 → 00:00 스케줄러가 보완 |
| 디버깅 난이도 | 이벤트 흐름이 코드 상 명시적이지 않아 추적이 어려울 수 있음 |
| 테스트 | `ApplicationEventPublisher` mock 처리 필요 |
