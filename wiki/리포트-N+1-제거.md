## STAR

### Situation

- 리포트 목록 API 호출 시 **주차 수(N)만큼** `weekly_safety_report` SELECT 반복
- 주차가 늘수록 응답 시간·DB 부하 선형 증가

### Task

- 목록 1회 조회에 필요한 safety signal을 **최소 쿼리**로 가져오기

### Action

- `ReportService.getReports()`에서 member의 safety report를 **일괄 조회**
- `associateBy { weekStart }`로 주차별 맵핑 후 리포트 DTO 조립

### Result

- 주차 N회 쿼리 → **1회 batch 조회**
- 리포트 주 수가 적을 때는 체감 작았으나, 구조적으로 N+1 제거
