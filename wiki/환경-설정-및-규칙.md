# 8. 환경 설정 및 규칙

## 로컬 실행

```bash
# DB만 Docker
docker-compose up -d db
./gradlew bootRun
# → http://localhost:8080

# DB + App Docker
docker-compose up -d
# → http://localhost:18080
```

## 필수 시크릿 (git 제외)

| 파일/변수 | 용도 |
|-----------|------|
| `src/main/resources/firebase-service-account.json` | Firebase Admin |
| `.env` | JWT, DB, AI URL, `INTERNAL_SERVICE_TOKEN` |
| `FIRESTORE_DATABASE_ID=bifriends` | Named Firestore DB |

예시: `deploy/.env.example`, `README.md`

## 배포 (GCP VM)

- [`deploy/DEPLOY.md`](https://github.com/quad-S-BIFriends/bifriends-be/blob/main/deploy/DEPLOY.md)
- 운영 URL: https://api.bifriends.study
- **주의:** CI는 이미지만 갱신 — VM의 `/app/.env`, `docker-compose.yml`은 **수동 동기화**

## 코딩 규칙 (요약)

| 항목 | 규칙 |
|------|------|
| Entity | `data class` 금지 (JPA) |
| API | `/api/v1` prefix |
| 시간 | `ZoneId.of("Asia/Seoul")` 명시 |
| 문서 | `bifriends-be/doc/` 하위 |
| 새 도메인 | Security 경로·Swagger 확인 |

상세: 레포 루트 `CLAUDE.md`, `README.md` 설계 규칙
