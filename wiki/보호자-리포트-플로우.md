## STAR

### Situation

- 보호자 리포트 생성·“지난 리포트 보러 가기” 등 **초기 플로우 오류**
- AI 리포트 생성은 **비동기** — 생성 직후 `sections` 미완 상태 가능

### Task

- 리포트 생성 요청 → AI 콜백 → DB 저장 → 조회 API **end-to-end 정합**
- FE가 기대하는 타이밍·필드 구조 맞추기

### Action

- 리포트 콜백 URL·payload 처리 정리
- `weekly_report` 저장·조회 API 플로우 수정
- 비동기 생성 중 상태(목록만 있고 본문 비어 있음) 문서화

### Result

- 초기 리포트 생성·조회 오류 해결 (#1 Closed)
- **남은 UX 이슈:** 같은 주 재생성 시 `reportId` 불변 → FE 폴링이 갱신 감지 못할 수 있음 (백로그)

## 운영 참고

| 시점 | 클라이언트 동작 |
|------|----------------|
| 생성 요청 직후 | 목록에 행만 있고 `sections` 비어 있을 수 있음 |
| AI 콜백 완료 후 | `GET /reports/{id}` 재조회 또는 폴링 |
| 같은 주 재생성 | `reportId` 동일 → 새 완료 감지 로직 필요 (FE) |
