# 2. 스택 및 아키텍처

## 기술 스택

| 구분 | 기술 |
|------|------|
| Language | Kotlin 2.1 / JDK 21 |
| Framework | Spring Boot 3.4 |
| DB | PostgreSQL 17, Flyway(운영) |
| ORM | Spring Data JPA |
| Auth | Firebase ID Token → JWT |
| BaaS | Firestore, Firebase Storage |
| AI 연동 | HTTP REST (`bifriends-ai`) |
| API 문서 | springdoc-openapi (Swagger UI) |
| 배포 | Docker Compose, GCP VM |

## 시스템 구성 (요약)

```
Flutter App
    │ Bearer JWT
    ▼
Spring Boot (bifriends-be)
    ├── PostgreSQL  … 회원·학습·채팅·리포트·상점
    ├── Firestore   … 친구랑 완료 세션
    └── Storage     … 감정 이미지
    │
    │ HTTP (내부망)
    ▼
bifriends-ai (Python)
    … Leo 채팅, 시나리오, 주간 리포트·안전 배치
```

## 패키지 구조

```
com.bifriends/
├── domain/{name}/     controller · service · repository · model · dto
├── infrastructure/    security, firebase, ai 클라이언트
└── global/            config, exception
```

- API prefix: `/api/v1`
- AI 내부 API: `X-Internal-Service` 헤더

## 데이터 저장소 분리

| 저장소 | 담당 데이터 |
|--------|-------------|
| PostgreSQL | 관계형·집계 (채팅 메시지, 학습 시도, weekly_report 등) |
| Firestore | 친구랑 세션 JSON, `learnedExpressions` 요약 |
| Firebase Storage | AI/폴백 감정 이미지 |

## 스케줄러

| 스케줄러 | 시각 (KST) | 역할 |
|----------|------------|------|
| `TodoScheduler` | 매일 00:00 | 할 일 3개 생성 |
| `WeeklyReportScheduler` | 월 01:00 | 주간 성장 리포트 AI 트리거 |
| `WeeklySafetyScheduler` | 월 01:00 | 챗 안전 신호 AI 트리거 |
